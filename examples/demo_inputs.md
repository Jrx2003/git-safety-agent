# Demo 输入示例

## 单次输入一种操作

- 看看当前仓库状态
- 查看最近提交历史
- 查看 git 差异
- 提交: 修复登录按钮
- 暂存所有改动
- 切换分支: develop
- 创建分支: feature/login
- 删除分支: old-branch
- 初始化 git 仓库

## 一次输入一系列操作

```
看看当前仓库状态
查看最近提交历史
暂存所有改动
提交: 修复登录按钮
切换分支: develop
```

```
初始化 git 仓库
创建分支: feature/auth
切换分支: feature/auth
```

## 混合危险指令（应被拦截但保留安全步骤）

```
看看当前仓库状态
提交: 修复登录按钮
请 reset --hard
```

```
查看最近提交历史
push --force
```

## 索引相关（计划执行模式）

```
构建索引并总结目录
搜索 认证 相关代码
给出目录整理建议
```

## 索引问答（切换到“索引问答”模式）

- 这个仓库是做什么的？请简要说明。
- 模型选择相关的代码在哪？请给出关键片段。
- llm_client.py 里是如何设置 base_url 的？
- 工具注册（MCP registry）在哪里？主要有哪些工具？
- 目录结构如何组织？有哪些模块？

## 澄清追问示例

- 我要提交代码（未给出提交信息）
- 暂存代码（未说明具体路径）
- 切换分支（未说明目标分支）
